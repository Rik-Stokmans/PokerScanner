# The Silent Table

A premium poker scanning and analysis Flutter app. Track live games, evaluate hands in real time, review session history, and learn poker fundamentals — all backed by Firebase.

## Features

- **Live poker table** — track hole cards and community cards with real-time hand evaluation (all ranks from high card to royal flush, including wheel straights)
- **Scanner setup** — configure the card scanning workflow before a session
- **Lobby** — create or join games and manage active sessions
- **Invitations & friends** — send/receive game invitations and manage your friends list
- **Session analysis** — post-session stats and charts via fl_chart
- **Game history** — browse past hands and results
- **Learn** — poker fundamentals reference

## Tech stack

| Layer | Library |
|---|---|
| UI | Flutter + Material 3 dark theme |
| Navigation | go_router 13 |
| State | flutter_riverpod 2 |
| Backend | Firebase Auth + Cloud Firestore |
| Charts | fl_chart |
| Fonts | google_fonts |

## Getting started

### Prerequisites

- Flutter SDK — project was built with the SDK at `~/App development/flutter`
- A Firebase project with Auth (email/password) and Firestore enabled

### Firebase setup

The Firebase config files are gitignored and must be generated locally:

```bash
dart pub global activate flutterfire_cli
flutterfire configure
```

This regenerates `lib/firebase_options.dart` and `android/app/google-services.json`. Run it once after cloning.

### Install and run

```bash
flutter pub get
flutter run
```

## Project structure

```
lib/
  main.dart                  # App entry point
  router.dart                # go_router routes + auth redirect guard
  firebase_options.dart      # Generated by flutterfire configure (gitignored)
  models/                    # Data models (User, Game, Hand, Card, ...)
  services/
    auth_service.dart        # Firebase Auth wrapper
    firestore_service.dart   # Firestore reads/writes
    hand_evaluator.dart      # Pure-Dart 7-card hand evaluator
    bot_service.dart         # Bot/AI opponent logic
  providers/                 # Riverpod providers
  screens/                   # One file per screen (10 screens)
  widgets/                   # Shared widgets
  theme/                     # Colors and ThemeData
```
